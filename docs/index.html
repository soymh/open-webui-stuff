GPT4-Memory Mimic Documentation

Overview

GPT4-Memory Mimic is designed to extend the functionality of a local large language model (LLM) web UI by incorporating a dynamic memory feature. This feature allows the LLM to retain, recall, and manage information from user interactions over time, enhancing context-aware responses similar to human-like conversation flow.

Key Developments

  • User-Controlled Memory Addition: Allows users to decide what to store in memory.
  • Automatic Important Data Storage: Identifies and stores crucial information.
  • Contradiction Handling in Memory: Updates inconsistent information automatically.
  • Integration with LLM for Summarization: Utilizes LLM to summarize inputs before saving.

Changes Applied

User-Controlled Memory Addition

Modified the add_to_memory_if_user_wants method to allow users to decide what gets stored in memory. This feature enhances user control over personal data retention.

Automatic Important Data Storage

Implemented the add_important_data method, allowing the system to identify and store key pieces of acquired information. This ensures crucial data is always retained and easily accessible.

Handling Contradictory Information

Added logic to identify and update contradictions in stored memory, maintaining the consistency and reliability of stored data.
